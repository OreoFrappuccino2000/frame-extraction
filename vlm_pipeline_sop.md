# VLM 能力提升管道 - 标准操作程序 (SOP)

## 概述

本 SOP 描述了 PUBG Mobile 视频分析管道的完整工作流程，从视频输入到最终的多语言字幕输出，重点在于 VLM 能力的持续提升。

## 系统架构

```
视频输入 → 帧提取 → VLM 分析 → Skills 校验 → 事件分段 → 多语言生成 → 音频合成 → 最终输出
```

## 1. 视频输入阶段

### 输入格式
- 支持 MP4、MOV、AVI 等常见视频格式
- 建议分辨率：1080p 或更高
- 建议帧率：25-30fps

### API 端点
```
POST /extract-frames
Content-Type: application/json

{
  "video_url": "视频URL或本地路径",
  "fps": 25,
  "max_frames": 500
}
```

### 输出
- 帧图像列表（Base64 编码）
- 每帧的时间戳映射
- 视频元数据

## 2. VLM 分析阶段

### VLM 系统提示词
VLM 使用专门的系统提示词进行 PUBG Mobile 游戏分析，重点提取：
- 战斗事件（击倒、淘汰、救援）
- 玩家状态（生命值、位置、动作）
- 团队动态和意图识别

### 持续改进策略
1. **数据收集**：每次分析后保存 VLM 输出样本
2. **错误分析**：识别常见的幻觉和误识别模式
3. **提示词优化**：基于错误分析调整系统提示词
4. **技能规则更新**：更新 skills.md 中的校验规则

## 3. Skills 校验层

### 功能特性
- **时间一致性校验**：确保事件时间戳准确
- **置信度过滤**：过滤低置信度事件（阈值：0.55）
- **意图识别**：检测连杀、救援、团队对抗等意图
- **玩家名称模糊化**：符合"老白"解说风格
- **防幻觉机制**：基于可见 UI 证据验证事件

### 集成端点
```
POST /validate-vlm
Content-Type: application/json

{
  "vlm_output": "VLM 原始输出",
  "frames": "帧提取结果"
}
```

### 输出增强
- 校验后的事件、视图、指标、小队数据
- 意图分析结果
- 详细的校验日志
- 玩家名称模糊化处理

## 4. 事件分段阶段

### 分段策略
- **Intro Hooks**：开场钩子，吸引观众注意
- **Global Filler**：过渡和铺垫内容
- **Call Events**：关键事件反应
- **Outro Punchlines**：收尾总结（可选）

### 智能分段规则
- 基于事件优先级和时间分布
- 考虑解说节奏和观众体验
- 支持动态调整分段策略

## 5. 多语言生成阶段

### 语言支持
- 中文（主要）："老白"风格解说
- 英文：专业电竞解说风格
- 其他语言：根据需求扩展

### 生成规则
- 每段 ≤ 2 句话，更偏 1 句话
- 自然口语化，避免专业术语
- 符合游戏解说节奏
- 字数限制：不超过 max_units

## 6. 音频合成阶段

### TTS 集成
- 支持多种 TTS 引擎
- 语音风格匹配解说内容
- 时间轴精确对齐

### 质量控制
- 音频时长计算和验证
- 语音自然度检查
- 时间戳同步验证

## 持续改进流程

### 每周迭代周期

#### 周一：数据收集和分析
- 收集上周的所有 VLM 输出样本
- 分析常见错误模式和幻觉
- 识别系统提示词的改进点

#### 周二：规则更新
- 更新 skills.md 中的校验规则
- 添加新的意图识别模式
- 优化玩家名称模糊化逻辑

#### 周三：提示词优化
- 基于错误分析优化 VLM 系统提示词
- 测试新的提示词变体
- 验证改进效果

#### 周四：集成测试
- 使用历史视频测试改进后的系统
- 比较新旧版本的输出质量
- 量化改进指标

#### 周五：部署和监控
- 部署改进到生产环境
- 监控新版本的表现
- 收集用户反馈

### 质量指标

#### VLM 质量指标
- **精确率**：正确识别的事件比例
- **召回率**：实际事件被识别的比例
- **幻觉率**：错误生成的事件比例
- **置信度分布**：事件置信度统计

#### 解说质量指标
- **自然度评分**：解说的自然流畅程度
- **相关性评分**：解说与视频内容的相关性
- **观众满意度**：用户反馈评分

## 故障排除指南

### 常见问题

#### VLM 输出质量差
1. 检查视频质量（分辨率、帧率）
2. 验证系统提示词是否最新
3. 分析具体的错误模式
4. 调整置信度阈值

#### Skills 校验失败
1. 检查 skills.md 文件格式
2. 验证时间戳映射是否正确
3. 检查玩家名称模糊化规则
4. 查看校验日志详情

#### 音频同步问题
1. 验证 TTS 时长计算
2. 检查时间轴对齐逻辑
3. 测试不同视频长度的处理

### 调试工具
- 使用 `/validate-vlm` 端点进行中间结果验证
- 查看详细的校验日志和意图分析
- 使用测试脚本验证特定功能

## 扩展和定制

### 添加新游戏支持
1. 分析新游戏的 UI 元素和事件类型
2. 更新 VLM 系统提示词
3. 扩展 skills.md 中的校验规则
4. 调整分段和生成策略

### 多语言扩展
1. 收集目标语言的解说样本
2. 训练或配置相应的 TTS 引擎
3. 调整生成规则以适应语言特点

### 性能优化
1. 并行处理多个视频段
2. 缓存常用计算结果
3. 优化图像处理流水线
